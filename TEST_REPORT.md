## 跨区域共识流程验证

**实际观测到的完整流程（ALI日志）**：
```
1. ALI 创建新区块 (495b06d6...)，数据=性能指标+本机数据
2. ALI 广播区块提议 → TC 节点
3. TC 收到提议，验证哈希，投票: 同意
4. ALI 自己也投票: 同意
5. 同意率 = 2/2 = 100% ≥ 80% 阈值 → 区块正式确认
6. ALI 写入本地链（高度+1）
7. TC 通过 /p2p/chain/sync 定期同步，获取 ALI 新块写入本地链
```

**轮询共识**：`ali` 和 `tc` 按字母排序轮流出块（ali→tc→ali→tc...）

## 已知限制（非Bug）

1. **IPMI FRU 采集失败**：云主机没有 BMC 控制器，ipmitool 无法连接是预期行为。生产环境须有真实 BMC 设备
2. **创世块哈希不同**：两节点独立启动时各自生成创世块，通过最长链原则自动同步。生产环境建议通过配置文件预置相同 `genesis_hash`
3. **P2P 仅支持白名单节点**：新节点加入需手动更新所有节点配置文件
